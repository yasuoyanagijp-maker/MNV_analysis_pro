# ARIAKE OCTA — ユーザーマニュアル

加齢黄斑変性症（MNV）および血管密度（VD）解析向けのデスクトップ／ブラウザ UI です。本書は**研究・臨床オペレーター**向けの操作説明です。

---

## 1. 概要

| 項目 | 内容 |
|------|------|
| アプリ名 | ARIAKE OCTA（Retinal Analysis Pro 系） |
| 主な用途 | OCTA 等の画像に対する **MNV 解析**、**VD 解析**（単一／バッチ） |
| フロント | Flet（Python） |
| API | FastAPI（既定で `http://127.0.0.1:8000` に接続） |

---

## 2. 動作環境

- **Python** 3.9 以上（プロジェクトでは 3.9+ 想定）
- **OS** 例：macOS（`run_flet.sh` は macOS 向けコメントあり）
- 仮想環境（`.venv`）への依存パッケージインストールが前提です。

---

## 3. セットアップ（初回）

1. リポジトリを取得し、プロジェクトルートに移動します。  
2. 仮想環境を作成・有効化し、依存関係を入れます。

```bash
python3 -m venv .venv
source .venv/bin/activate   # Windows は .venv\Scripts\activate
pip install -r requirements.txt
```

---

## 4. 起動方法

### 4.1 推奨：一体起動スクリプト（バックエンド + UI）

```bash
./run_flet.sh
```

スクリプトは次を行います。

- ポート **8000**（API）と **FLET_PORT**（既定 **8550**、Flet）の空き確認
- **FastAPI** をバックグラウンドで起動（ログは `backend.log`）
- **Flet** フロントを起動

### 4.2 手動起動

ターミナルを二つに分ける例です。

**API：**

```bash
source .venv/bin/activate
python src/api/main.py
```

**UI：**

```bash
source .venv/bin/activate
flet run --web --port 8550 main_app.py
```

または `python main_app.py`（`FLET_USE_WEB` により挙動が変わります。次節）。

### 4.3 ブラウザで開く

`FLET_USE_WEB=1`（既定）のとき、**Web ブラウザ**で UI が開きます。表示されない場合は、`http://127.0.0.1:8550`（ポートは `FLET_PORT` で変更可）を開いてください。

---

## 5. 表示モード：Web とネイティブ

Flet では **ブラウザ（Web）** と **ネイティブウィンドウ** を切り替えられます。環境変数 **`FLET_USE_WEB`** で統一しています。

| 設定 | 意味 |
|------|------|
| `FLET_USE_WEB=1`（省略時と同じ） | **ブラウザ**で表示。OS のフォルダ選択ダイアログ（`get_directory_path`）は**使えません**（Flet の制限）。 |
| `FLET_USE_WEB=0` または `native` 等 | **ネイティブウィンドウ**。OS のファイル／フォルダダイアログが利用可能な場合があります。 |

**例（ネイティブで起動）：**

```bash
FLET_USE_WEB=0 ./run_flet.sh
```

**例（ポート変更）：**

```bash
FLET_PORT=9000 ./run_flet.sh
```

Web モードでは、フォルダ／画像の指定に **手動パス入力**や、**API が参照できるパス**での**サーバー側フォルダ一覧**（アプリ内のエクスプローラ）が補助として使われます。解析対象データは、**API を動かしているマシン上**（またはマウントされたパス）に置く必要があります。

---

## 6. ログイン

- 通常起動時は **`/login`** から開始し、**研究者名**と**パスワード**を入力します。
- ログイン成功後、ホーム（`/`）へ遷移します。
- API の `/login` と連携します。接続できない場合はエラーメッセージを確認してください。

### 開発者向け注意（DEV_MODE）

環境変数 **`DEV_MODE=1`** を付けて起動すると、**ログインがバイパス**され、セッションに開発用ユーザーが入ります。**本番・共有端末では絶対に使わないでください。**

---

## 7. 画面とナビゲーション

ログイン後、左の**サイドバー**から主要画面へ移動できます。

| アイコン（目安） | 遷移先 | 内容 |
|------------------|--------|------|
| ホーム | `/` | **ダッシュボード**（解析種別・パス指定・バッチ） |
| 解析 | `/` または解析フロー | MNV / VD の各ステップ（`/roi`・`/mnv` など） |
| 結果 | `/results` | 解析結果の表示 |

ルートは URL パスと対応しています（例：`/results`）。

---

## 8. ダッシュボード（ホーム）

### 8.1 解析タイプ

- **MNV Analysis** — 単一画像の MNV 解析（ROI 設定へ進む場合があります）。
- **VD Analysis (Single)** — 単一画像向け VD。
- **VD Analysis (Batch Folder)** — フォルダ単位のバッチ。

**Image Scale (mm)** は画像スケール設定として利用します（解析 API に渡されます）。

### 8.2 画像・フォルダの指定

- **Select Folder / Select Image**  
  - **ネイティブ**：OS のダイアログで選べる場合があります。  
  - **Web**：サーバー側パスを選ぶ UI や、**手動パス**が主な手段です。
- **Manual Path**  
  ファイルまたはフォルダの**絶対パス**を貼り付けて利用できます。引用符（`'` `"`）は自動で除かれることがあります。
- **ドラッグ＆ドロップ**  
  対応している場合、ドロップしたパスで処理が始まります。

### 8.3 バッチ解析

フォルダ内の対応画像を一覧に載せ、**Start Batch Analysis** でキュー実行します。完了後、**結果画面**へ進む動作になります。

**対応拡張子の例**（ダッシュボードの走査に基づく）：`.tif` `.tiff` `.png` `.jpg` `.jpeg`

---

## 9. ROI 画面（`/roi`）と MNV ウィザード（`/mnv`）

- **MNV** 系のフローでは、画像と解析設定に応じて **ROI（関心領域）** の指定やウィザード画面が表示されます。
- 画面の指示に従い、**戻る／次へ**や**解析開始**を行ってください。

---

## 10. 結果画面（`/results`）

- 直近の解析結果やバッチ結果のサマリーを閲覧します。
- データ量が多いと表示に時間がかかる場合があります。

## 11. トラブルシューティング

| 現象 | 確認すること |
|------|----------------|
| ブラウザが開かない | `FLET_PORT` が他プロセスと競合していないか、ファイアウォールを確認。 |
| API に接続できない | バックエンドが起動しているか、`127.0.0.1:8000` に届くか。 |
| Web でフォルダが選べない | **仕様上、OS のフォルダダイアログは使えません。** 手動パス、サーバー側エクスプローラ、または **`FLET_USE_WEB=0`** でネイティブ起動を試す。 |
| パスが通らない | パスが **API プロセスから見える**か（共有フォルダのマウント、権限）。 |
| ログインできない | `DEV_MODE` ではないことを確認。API の認証とネットワークを確認。 |
